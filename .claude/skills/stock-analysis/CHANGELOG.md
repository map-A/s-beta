# 本次更新说明

## 没有变的部分
- `SKILL.md` 的9步主流程（获取F10数据 → 识别股票信息 → Serenity 7步分析 → 读取模板 → 生成4个输出文件 → 验证输出）结构完全不变。
- 所有外部脚本路径、模板路径、输出目录结构、JSON字段规范（categories/symbolSize/relation取值等）保持不变。
- Serenity「卡点」四项标准的定义本身不变。

## 改了什么、为什么

1. **`SKILL.md` 头部描述和开场白**：原来写的是"精通半导体和AI硬件供应链"，这会让Claude在遇到非半导体行业的股票时不太愿意触发这个技能，也容易把分析思路锚定在算力供应链的具体例子上。改成了通用表述，并明确"半导体只是一个应用案例，不是边界"。

2. **新增 `references/methodology.md`（方法论通用化）**：原来的方法论散落在 `SKILL.md` 正文里，且举例全部是半导体。这次把方法论本身（四项标准的判断逻辑、七步流程的设计原理、仓位管理的底层逻辑、常见误判模式、方法论的适用边界）单独抽出来，每一条都配了半导体之外的例子（新能源电池、创新药CDMO、军工材料、种业、稀土磁材等），这样Claude在分析新行业时有方法论可以套用，而不是只能模仿半导体案例。

3. **重写 `references/chain_map.md`（产业链映射，原文件内容质量不高，本次基本重写）**：
   - 新增"第0节：通用建模方法"——遇到模板没覆盖的新行业（比如人形机器人、低空经济、合成生物学）时，教Claude自己用5步法画产业链图，而不是卡死在已有模板里。
   - 把原来只有AI硬件一个行业的内容，扩展到10个行业：半导体/AI算力、新能源车电池、光伏风电、军工航空、创新药CXO、农业种业、稀土磁材、特种化学品新材料、消费电子、航运物流。每个行业给出L0-L4层级、典型卡点定位、A股概念板块映射。
   - 新增"第11节：适用边界"——明确哪些行业（白酒/消费品、金融、互联网平台等）**不**适合用卡点框架硬套，避免Claude给品牌驱动型公司也强行打卡点星级。

4. **重写 `references/verification.md`（验证标准，原文件内容质量不高，本次基本重写）**：
   - 新增"confirmed / inferred / rumor"三档证据等级的清晰判定标准和写作措辞要求，避免分析文档里把"推测"写成"确定事实"。
   - 新增分行业的官方验证渠道对照表（医药查NMPA批文、种业查农业农村部品种审定公告、军工查国防科工局中标公告等），原来只有半导体一套验证习惯。
   - 新增"常见造假/误导信号"清单，泛化到所有行业的关联交易包装、概念蹭热点、业绩对赌冲刺等模式。

5. **重写 `references/cjk_addendum.md`（A股特色，原文件内容质量不高，本次基本重写）**：
   - 国产替代政策周期框架从"半导体专属"泛化为四阶段通用框架，并给出不同行业对应的政策文件/部委类型对照表。
   - A股特有风险大全（商誉/质押/关联交易/业绩对赌/财务造假/ST退市）按行业补充了各自特有的合规风险类型（医药的临床数据造假、军工的保密资质取消、农业的转基因违规推广等）。

6. **重写 `references/supply_chain_graph.json`（原文件内容质量不高，本次重新设计schema）**：
   - 原来是单一半导体/AI硬件的12层供应链图。改成了 `industries` 多行业分组结构，每个行业下有 `layers` / `confirmed_supply_edges` / `chokepoint_candidates`。
   - **重要变化**：`confirmed_supply_edges` 里不再写"某公司向某公司供货"这种具体到公司名字、容易过时或失实的断言，改成角色级别的关系（比如"稀土分离冶炼厂 → 永磁材料厂"），并在每条edge上标注 `confidence: "industry_pattern"`，意思是"这是行业公开的结构性逻辑，不是对当前某家公司的事实确认"。具体公司级别的关系，需要在分析时结合F10真实数据生成假设，再按 `verification.md` 的三档标准核实，这样不会让Claude把这个JSON里的示例当成"已验证的真实供应商关系"直接抄进分析报告。

7. **`SKILL.md` 正文里散落补充了跨行业提示**：比如步骤1的"信号源"举例增加了新能源车/医药/军工/农业各自该跟踪谁，步骤4的"一手验证"来源增加了不同行业该看哪个数据源，步骤6补充了"军工等行业信息差更多来自啃公告的苦功夫而非内幕优势"。这些都是在不改变7步骨架的前提下，让每一步在新行业里有具体可执行的抓手，而不是空喊"跨行业也适用"。

## 关于具体公司/数据的免责声明

新写的几个参考文件里，凡是提到具体公司名字的地方，都标注成"历史模式参考/行业结构示例"，并要求在写入正式分析文档前用 `fetch_f10_data.py` 和 WebSearch 重新核实。这是因为我（Claude）没有实时数据访问能力，也不应该在知识库里写入看起来像"已验证事实"但其实是我训练数据里可能过时或不准确的具体公司财务关系——这样设计是为了让整个技能体系本身的"诚实标注不确定性"原则也适用于知识库文件本身，不只是适用于最终输出的分析报告。

## 如何使用这次更新

1. 把这个包里的 `SKILL.md` 覆盖到 `~/.claude/skills/stock-analysis/SKILL.md`。
2. 把这个包里 `references/` 目录下的5个文件覆盖到 `~/.claude/skills/stock-analysis/references/`。
3. 其他文件（`fetch_f10_data.py`、`templates/` 下的3个模板）不受影响，不需要改动。
4. 建议先用一个非半导体的股票（比如某新能源车电池材料公司，或者某创新药CDMO公司）跑一遍流程，看看Claude是否正确识别了行业模块、读取了对应的chain_map.md章节，再决定要不要进一步调整某个行业的层级模板细节。

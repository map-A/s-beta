## 1. 证据等级金字塔

由强到弱排列,分析时优先使用上层证据,下层证据只能作为补充和交叉验证,不能单独作为结论依据:

```
第一级(最强): 法定披露文件
  年报/半年报/招股说明书 的客户供应商披露、政府监管批文(药品批文/品种审定证书/
  军品定型文件)、工商登记/股权变更公告、交易所问询函回复

第二级: 官方/权威数据源
  政府部门公示(工信部/药监局/农业农村部/国防科工局等)、行业协会统计数据、
  海关进出口数据、大宗商品现货价格指数(如上海有色网、百川盈孚)

第三级: 机构与媒体研究
  券商研报(注意研报立场可能偏多)、行业会议纪要、財经媒体深度报道、
  上市公司投资者关系活动记录(互动易/投资者关系活动表)

第四级(最弱,仅供参考方向,不可单独作为结论): 
  自媒体/股吧/社交媒体传言、未经核实的"业内人士"转述、
  匿名社交平台的"小道消息"
```

**铁律**:任何进入最终分析文档的具体论断,至少要有第一级或第二级证据支持;仅有第三、四级证据时,必须在文档中用"据XX报道,尚未经一级证据确认"等措辞明确标注,不能写成确定性语言。

## 2. 三档判定标准（confirmed / inferred / rumor）

| 等级 | 判定标准 | 写作措辞要求 |
|---|---|---|
| **confirmed(已确认)** | 有第一级证据明确披露的关系/事实,且在最近1-2期报告中仍然有效 | 直接陈述,可注明披露来源和报告期 |
| **inferred(推测)** | 基于二跳逻辑、行业常识或第二/三级证据合理推断,但无直接披露文件确认 | 必须使用"推测""可能""据XX间接证据推断"等措辞 |
| **rumor(传言)** | 仅有第四级证据,或来源单一且无法交叉验证 | 仅作为研究线索记录,不写入正式结论;如确需提及,标注"未经证实的市场传言" |

## 3. 分行业验证路径

不同行业的"权威批文/资质文件"不同,分析时要去对应的官方渠道核实,而不是套用半导体的验证习惯:

| 行业 | 关键权威验证渠道 |
|---|---|
| 半导体/电子 | 客户认证公告、海关进出口数据、SEMI/行业协会产能数据 |
| 创新药/医疗器械 | 国家药监局(NMPA)药品/器械批文查询、ClinicalTrials.gov/药物临床试验登记平台、医保谈判结果公示 |
| 农业/种业 | 农业农村部品种审定公告、植物新品种权公告、全国农技中心推广面积统计 |
来源标注示例:"根据全国农业技术推广服务中心年度推广面积数据" |
| 军工/航空航天 | 国防科工局/装备采购信息网中标公告、上市公司"军品资质"披露(如保密资质、武器装备质量管理体系认证) |
| 新能源/电池 | 工信部新能源汽车推广目录、动力电池白名单、车企供应商定点公告 |
| 稀土/矿产资源 | 工信部稀土开采/冶炼分离总量控制指标公告、自然资源部矿产资源储量数据 |
| 光伏/风电 | 中国光伏行业协会(CPIA)产能与价格数据、国家能源局新增装机统计 |
| 化工/新材料 | 安全生产许可证、危化品经营许可、行业协会(如中国化工学会)产能统计 |

## 4. 多源交叉验证矩阵

对任何一条"供应链关系"结论,建议至少填一遍下表,凡是只有单一来源支持的结论,默认降级为"inferred":

```
结论: [公司A 向 公司B 供应 XX产品/服务]

来源1(类型/等级): ___________  →  支持/不支持/部分支持
来源2(类型/等级): ___________  →  支持/不支持/部分支持
来源3(类型/等级): ___________  →  支持/不支持/部分支持

最终判定: confirmed / inferred / rumor
判定理由: ___________
最近核实日期: ___________
```

## 5. 常见造假/误导信号（跨行业）

- **关联交易包装成"供应链合作"**:核实交易对手是否为关联方(查股权结构/实控人关联关系),关联交易额占比异常高的"供应商/客户"关系需要打折看待。
- **战略合作公告 ≠ 实质订单**:签署"战略合作协议""战略框架协议"只是意向,要核实是否有具体订单金额、交付时间表,以及后续季报是否体现为收入。
- **蹭概念式更名/跨界**:公司主营业务与新概念关联度低,但通过更名、设立子公司、参股小比例公司来蹭热点,需核查实际收入占比。
- **政府补贴/政策红利期的虚高毛利率**:核实毛利率改善是来自真实成本/技术优势,还是来自一次性补贴、税收优惠到期前的窗口期。
- **业绩对赌到期前的"业绩冲刺"**:并购重组中常见,对赌到期当年业绩异常亮眼,次年大幅下滑甚至商誉减值,需关注业绩承诺到期时间表。
- **样品/小试当量产宣传**:任何行业,"研发取得突破""通过XX测试"都需要追问是实验室级别还是量产级别,是否已形成稳定良率和规模交付能力。

## 6. 信息时效性管理

卡点关系不是静态的,以下情况会导致历史"confirmed"结论失效,需要定期复核(建议至少每个财报季复核一次相关持仓的供应链关系):

- 下游客户引入第二供应商,集中度下降；
- 上游环节出现新进入者通过认证；
- 行业技术路线发生迁移(例如电池技术路线切换、先进制程节点切换),原卡点环节可能被新路线绕开；
- 监管政策变化(集采、出口管制调整、关税政策)改变了原有格局；
- 公司自身资本运作(并购、分拆、股权变更)改变了原供应链关系的归属主体。

任何引用了具体数据的历史案例(包括本技能 `chain_map.md` 和 `supply_chain_graph.json` 中的示例),在用于实际分析前都必须重新核实当前是否仍然成立,不能直接复用历史结论。
